This document restructures the ESG platform requirements by user role, providing clear dashboard specifications, access functions, and workflow guidance for designers and developers building each role's interface.
1. Executive Summary
The ESG Data Collection Platform is a web-based system that replaces a manual, Excel-based process for collecting ESG (Environmental, Social, and Governance) data across a private equity portfolio of approximately 500 portfolio companies.
This document is restructured around three primary user roles, providing designers and developers with precise specifications for each role's dashboard, access functions, and workflows. Additional use cases (third-party auditors, due diligence, etc.) are captured separately in the Additional Use Cases section.

2.4 Key Workflows — Admin
2.4.1 Annual Cycle Initiation (ESG Manager)
Log into Admin Review Hub with Manager credentials.
Navigate to Data Collection Cycles → Create New Cycle.
Set cycle year, open date, submission deadline, and optional extension date.
Select template type per asset class (private equity / real estate / debt).
Review and configure ESG template — add, modify, or remove data points as needed. Ensure all points are tagged to regulatory frameworks.
Configure prefilled data carried forward from prior year validated submissions.
Set company-specific deadlines if required.
Configure automated reminder schedule (30, 14, 7, and 1 day before deadline).
Activate cycle — this triggers automatic company notification emails.
2.4.2 Reviewing a Company Submission (ESG Analyst)
Log in to Admin Review Hub.
Open Submissions dashboard. Filter by status: Submitted / Flagged.
Select a company to open the Submission Detail View.
Review automated validation flags. Add manual notes where required.
If data is acceptable → Mark as Reviewed → passes to Manager for approval.
If data needs correction → Request Resubmission → enter specific feedback → company is notified automatically.
2.4.3 Final Approval (ESG Manager)
Review submissions marked as Reviewed by the Analyst.
Approve (status → Approved, feeds into reporting layer) or Reject back to Analyst with comments.
Monitor overall cycle progress on the main dashboard.

3.4 Authentication & Access Configuration
LP users access the platform via secure login with unique credentials (MFA not required by default, configurable).
Each LP account is configured by the ESG Manager with a specific portfolio scope: full portfolio view or fund-specific view.
LPs can only see portfolio-level aggregated data unless the ESG Manager has explicitly authorised individual company-level access.
LP access is controlled independently per LP — one LP may see the full portfolio while another sees only Fund II data.
All LP activity (report downloads, dashboard views) is logged in the audit trail.
Session timeout applies after a defined period of inactivity.

For funds requiring third-party ESG data assurance, an optional auditor verification workflow can be enabled. This is an additional role separate from the three primary roles.

Auditor Functions
ESG Manager grants external auditor access to selected company submissions via a time-limited, read-only login.
Auditor reviews: submission data, data confidence flags, supporting documents, and automated validation results.

5.2  Additional Use Case: Due Diligence & Pre-Acquisition ESG Assessment
This module supports pre-acquisition ESG assessments of potential investments by the deal team, outside the standard annual data collection cycle.
Pre-Acquisition Workflow
ESG Analyst creates a new Due Diligence Assessment record in the platform and invites the target company's nominated contact.
Target company receives a focused ESG questionnaire (40–60 questions) covering material ESG risks and policy status.
Responses are validated in real time and submitted to the ESG team.
ESG Analyst reviews responses and generates a Pre-Acquisition ESG Risk Summary report shared with the deal team.
Report identifies material ESG risks, policy gaps, and recommended actions — informing investment decision-making.
If acquisition proceeds, pre-acquisition assessment data is carried forward to seed the company's first annual submission.
New Company Onboarding Workflow
ESG Manager creates a new company profile: company name, asset class, sector, geography, and designated ESG contacts.
System generates login credentials and sends welcome email to the designated contact.
New company completes a Baseline ESG Maturity Assessment covering current ESG policies, processes, and baseline metrics.
ESG Analyst generates an ESG Maturity Report identifying improvement areas and recommended action plan items.
Company is enrolled in the standard annual data collection cycle from the following cycle year.
If pre-acquisition assessment was completed, the baseline is pre-filled from that data to reduce duplication.
5.3  Additional Use Case: ESG Improvement Action Plans
Beyond data collection, the platform supports active ESG management through improvement action plans. This is a year-round feature, not limited to the submission window.
Company Action Plan Features
Each portfolio company can maintain an active ESG Action Plan within the platform.
Actions can be linked to specific ESG metrics or policy gaps identified during annual review.
Each action includes: title, description, target metric/area, owner name, target completion date, and status (Not Started / In Progress / Completed / Overdue).
Companies update action status at any time during the year — not just at submission time.
Completed actions are highlighted in the company's ESG report as evidence of continuous improvement.
Admin View of Action Plans
ESG Analysts can view all portfolio companies' action plans in a consolidated view.
Filter by status, asset class, sector, or metric area to identify portfolio-wide trends.
Action plan completion rates are included in LP reports and portfolio ESG dashboards.
ESG Manager can set recommended action plan items for all portfolio companies after each annual review cycle.
5.4  Additional Use Case: Exit Readiness ESG Pack
An auto-generated multi-year ESG performance summary for any portfolio company — suitable for M&A due diligence processes and vendor data rooms.
Triggered by ESG Manager or Deal Team User for a specific portfolio company.
Automatically compiles: ESG score trend (multi-year), policy compliance status, key environmental metrics, WHS record, governance summary, and data quality assessment.
Exportable in PDF format for direct inclusion in vendor due diligence data rooms.
No additional data entry required — generated from existing approved submission history.

7. Non-Functional Requirements
These requirements apply across all three roles and the entire platform.
Requirement
Specification
Performance
Support up to 500 concurrent users during peak submission periods without degradation in response time.
Availability
Target uptime of 99.5% or greater, excluding scheduled maintenance windows.
Security
All data encrypted in transit (TLS 1.2+) and at rest. Role-based access control (RBAC) enforced throughout. MFA required for all Admin role users.
Scalability
Capable of scaling to support additional portfolio companies, data points, or asset classes without architectural changes.
Auditability
Full audit log of all user actions, data submissions, edits, approvals, and system events. Logs retained for a minimum of 7 years.
Usability
Company portal must be intuitive for non-technical users to complete submissions without training. Supported by inline guidance and help text throughout.
Data Retention
All submitted ESG data retained for a minimum of 10 years for historical trend analysis and regulatory requirements.
Browser Compatibility
Must function on current versions of Chrome, Edge, Firefox, and Safari. Accessible on desktop and tablet devices.
Data Isolation
Company users must be strictly isolated to their own company's data. Enforced at the API layer, not only at the UI layer.
